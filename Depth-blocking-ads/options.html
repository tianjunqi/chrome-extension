Content Filter rules


README:

每行一个（One line per line）.
url`type$strReg~.....`outTime
其中`$~为插件预留字符,不能在后续使用.
url 为要过滤的页面url path or domain信息，也可以使用*匹配所有页面，支持使用正册
type 为替换的内容类型（obj、text、textad）； obj为页面dom对象，text为页面html内容

outTime 延迟执行毫秒
type 为text时;
一种为常规模式，strReg 为要被拦截的内容，当type为obj时只能使用class=xxx或id=xxx；可使用正册匹配页面内容
第二种模式为: text$str1$str2$number
str1为第一次要查找的字符位置，找到str1后，再从str1字符的位置再继续查找str2字符的位置， 找到str2字符后再从str2字符位置开始截取后面number位字符， 根据截取到的字符串中提取id和class标签属性值，再根据标签属性（class和id）获取的html对象，并删除此对象。


第三种模式为：class=xxx&tagName=xxx&textContent=xxx&parent=2
class=xxx查找对象className，也也可以使用id=xxx代替
tagName=xxx要查找的标签tagName，例如：tagName=span
textContent=xxx要查找标签中包括的内容，例如：textContent=广告
parent=2，根据上面获取到的对象，再继续往上获取父对象，例如parent=2，为获取父级的父级对象。
